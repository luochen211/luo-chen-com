# 每日单词小程序页面风格指南

> 版本：v1.0  
> 适用范围：`daily-words-miniprogram` 现有页面及后续新增页面  
> 依据：2026-07-18 的首页、学习页、挑战页、周报页与公共导航组件源码

## 1. 风格定位

这套小程序的核心视觉方向是：**温暖、轻松、有成长感的学习陪伴工具**。

它不强调考试压力，也不追求工具产品常见的冷硬效率感。页面以米白背景、森林绿主色、柔和的大圆角卡片构成，用黄色提示连续学习、收藏和阶段成就，让用户感受到“每天完成一点”的正反馈。

### 1.1 风格关键词

- **自然**：森林绿、嫩绿、小苗等元素传达积累与成长。
- **温和**：米白底色、低对比边界和柔和阴影降低视觉压力。
- **专注**：每个页面只保留一个主要任务，主按钮始终清晰。
- **鼓励**：连续天数、进度环、正确反馈与周报共同形成成就感。
- **亲切**：圆润形状和口语化文案让学习过程更像陪伴，而非考核。

### 1.2 一句话设计原则

> 像一株每天长一点的小苗：明确、轻松、持续给用户正反馈。

## 2. 视觉系统

### 2.1 颜色

#### 核心色板（现有代码）

| 角色 | 色值 | 用途 |
| --- | --- | --- |
| 品牌绿 | `#087443` | 主操作、选中态、关键词、进度与品牌识别 |
| 深绿 | `#075332` | 深色文字、渐变终点、强调层级 |
| 柔和绿 | `#E9F5EC` | 次级按钮、字母标记、轻量反馈背景 |
| 成就黄 | `#FFD452` | 连续学习、奖励、收藏与成就提示 |
| 主文字 | `#121B17` | 标题和正文 |
| 次文字 | `#7B817E` | 辅助说明、计数、提示语 |
| 页面底色 | `#F8F7F2` | 全局背景，避免纯白带来的生硬感 |
| 卡片白 | `rgba(255,255,255,.94)` | 卡片和浮层背景 |

#### 功能色（建议统一）

| 角色 | 建议色值 | 用途 |
| --- | --- | --- |
| 成功 | `#087443` | 正确、完成、已掌握 |
| 警示/待复习 | `#D73A32` | 错误、不认识、需要复习 |
| 警示浅底 | `#FFF0EC` | 待复习按钮或错误提示背景 |
| 分割线 | `#E2E1DD` | 进度槽、分隔线、弱边界 |
| 禁用文字 | `#A8ACA9` | 不可操作状态 |
| 禁用背景 | `#ECEDEA` | 不可操作控件 |

使用约束：

- 单个页面的高饱和强调色不超过两种，默认是品牌绿与成就黄。
- 红色只表达错误、风险或待复习，不作为装饰色。
- 大面积背景优先使用米白或柔和绿，不使用纯绿铺满整页。
- 正文必须保持足够对比度，避免用浅灰承载关键信息。

### 2.2 字体与文字层级

全局字体沿用系统字体：

```css
font-family: -apple-system, BlinkMacSystemFont, "PingFang SC", sans-serif;
```

英文单词主体可使用 `Georgia, serif`，形成学习内容与界面文字的区分。

| 层级 | 建议字号 | 字重 | 典型场景 |
| --- | --- | --- | --- |
| 单词主标题 | `64–68rpx` | 700 | 学习页英文单词 |
| 数据英雄数字 | `56–72rpx` | 700–800 | 连续天数、今日进度 |
| 页面标题 | `32–44rpx` | 700–800 | 首页标题、自定义导航标题 |
| 问题/模块标题 | `34–38rpx` | 700 | 挑战题目、报告结论 |
| 正文 | `27–31rpx` | 400–600 | 释义、选项、例句 |
| 辅助文字 | `22–26rpx` | 400 | 日期、提示、计数、标签 |
| Tab 标签 | `20rpx` | 400–600 | 底部导航 |

排版规则：

- 中文正文行高建议为 `1.5–1.7`，例句使用 `1.7`。
- 数据与单位形成层级差，不让单位抢占数字视觉重量。
- 一个模块内粗体层级不超过两档。
- 不使用全大写英文作为中文界面的常规标题。

### 2.3 间距与栅格

页面采用单列布局，以 `32rpx` 作为左右安全边距。

建议将间距收敛到以下刻度：

| Token | 数值 | 用途 |
| --- | --- | --- |
| `space-1` | `8rpx` | 图标与短标签的紧密间距 |
| `space-2` | `16rpx` | 同组文字、标签间距 |
| `space-3` | `24rpx` | 控件内部或紧凑模块间距 |
| `space-4` | `32rpx` | 页面边距、卡片内边距 |
| `space-5` | `48rpx` | 主要模块之间 |
| `space-6` | `64rpx` | 页面大区块之间 |

约束：

- 卡片左右内边距优先使用 `30–36rpx`。
- 同类列表项间距保持一致，不为单个页面临时发明新间距。
- 固定底部导航的页面必须预留底部安全区和至少 `150rpx` 内容空间。

### 2.4 圆角、边界与阴影

整体形状语言是“大圆角、轻阴影、少描边”。

| 组件 | 圆角建议 |
| --- | --- |
| 主按钮 | `56rpx`，胶囊形 |
| 大卡片 | `30–34rpx` |
| 列表/选项卡片 | `24–26rpx` |
| 次级按钮 | `36–40rpx` |
| 小标签/状态点 | 圆形或完整胶囊 |

基础卡片阴影沿用：

```css
box-shadow: 0 10rpx 30rpx rgba(59, 58, 48, .09);
```

主操作或品牌卡片可略强，但阴影透明度不超过 `.24`。边框仅用于选中态、输入态或必要的结构区分，普通卡片不同时叠加明显描边和重阴影。

## 3. 核心组件规范

### 3.1 自定义导航栏

- 高度：现有实现为 `176rpx`，包含顶部安全区。
- 标题居中，`32rpx / 700`。
- 返回与右侧动作保持最小 `64rpx` 可点击高度。
- 导航动作优先使用统一线性图标，不使用含义模糊的字符替代。
- 首页可省略返回键；二级页必须提供明确返回路径。

### 3.2 主按钮

- 全宽、`112rpx` 高、`56rpx` 圆角。
- 使用绿色渐变 `#158B59 → #006B3B`，白色粗体文字。
- 一个页面默认只有一个主按钮。
- 按下态：缩放至 `.985` 并轻微降低透明度。
- 文案使用动词开头，例如“开始学习”“下一题”“查看报告”。

### 3.3 次级与语义按钮

- 次级正向操作：柔和绿背景 + 品牌绿文字。
- 负向/待复习操作：浅红背景 + 红色文字。
- 并列按钮必须在尺寸、圆角和文字结构上对称。
- 不只依赖颜色表达含义，应同时提供文字或图标。

### 3.4 卡片

卡片用于承载一个完整任务或一组强相关信息：

- 白色卡片：常规内容，如今日目标、单词详情、答题选项。
- 绿色渐变卡片：核心状态，如连续学习。
- 柔和绿卡片：总结、鼓励或阶段结果。
- 卡片内部遵循“标题/状态 → 主内容 → 辅助说明 → 操作”的顺序。

避免为了视觉丰富把每一段内容都装进卡片；同一区块信息若无需独立操作，应保持在同一容器中。

### 3.5 进度表达

当前存在三种进度形式，各自用途明确：

- **环形进度**：展示“今日目标”这类单一总体完成度。
- **横向进度条**：展示答题流程中的当前位置。
- **七日节点/柱状图**：展示时间维度上的连续性与趋势。

进度组件必须同时显示数字，不能只依赖图形估读。动画时长建议控制在 `600–900ms`，只在首次进入或数据变化时播放。

### 3.6 底部导航

- 固定底部，白色半透明背景，顶部使用极浅分割线。
- 图标建议 `40–44rpx`，标签 `20rpx`。
- 默认态为中性灰，选中态为品牌绿。
- 图标统一使用同一套线性图标，不混用 Emoji、实心图标与不同笔画风格。
- 当前 MVP 展示 4 个入口；未实现页面在正式版本中不应保留可点击假入口。

## 4. 页面结构模板

### 4.1 首页：状态驱动

信息顺序：

1. 今日身份与入口标题。
2. 连续学习状态，强化习惯与成就。
3. 今日目标和完成度。
4. 唯一主操作“开始学习”。
5. 底部全局导航。

首页首屏应回答三个问题：我坚持了多久、今天完成了多少、下一步做什么。

### 4.2 学习页：内容聚焦

信息顺序：

1. 当前学习进度。
2. 单词、音标与发音。
3. 释义和例句。
4. 收藏、认识/不认识等判断操作。
5. 手势提示等低优先级说明。

单词卡是页面唯一视觉中心。学习判断按钮保持在拇指容易触达的位置，并确保“认识”和“不认识”不只通过表情区分。

### 4.3 挑战页：逐步决策

信息顺序：

1. 题目进度。
2. 问题。
3. 单选项列表。
4. 选择反馈。
5. 下一步操作。

未选择答案前，“下一题”应禁用；选择后显示明确的正确/错误反馈。错误反馈应同时告诉用户正确答案，而不是只标红。

### 4.4 周报页：成就总结

信息顺序：

1. 一句结论与鼓励。
2. 一周趋势图。
3. 核心指标摘要。
4. 分享或继续学习的操作。

报告页允许使用更多成就黄，但仍以品牌绿为主。数据必须配标签和时间范围，避免只有装饰性图表。

## 5. 图标、插画与动效

### 5.1 图标

- 推荐统一使用 Lucide 风格线性图标，常规线宽保持一致。
- 图标默认继承文字颜色，选中态使用品牌绿。
- Emoji 可用于极少量情绪表达，不用于导航、收藏、发音、返回等功能性操作。
- 任何仅图标按钮都必须配置可访问名称。

### 5.2 插画

小苗是当前最有辨识度的视觉母题，适合继续用于：空状态、连续学习、升级成长和新手引导。插画应保持几何简洁、色块柔和，不引入写实图片或复杂 3D 风格。

### 5.3 动效

- 进入动画只服务于进度增长、数据出现或状态变化。
- 按钮反馈控制在 `100–180ms`。
- 图表与进度动画控制在 `600–900ms`。
- 小苗呼吸等环境动效可使用约 `3s` 的缓慢循环。
- 遵循减少动态效果设置；关键状态不可只通过动画传达。

## 6. 文案语气

文案应像温和的学习伙伴：短、明确、具体，不制造焦虑。

| 场景 | 推荐 | 避免 |
| --- | --- | --- |
| 行动 | 开始学习 | 立即开启你的逆袭之路 |
| 未完成 | 再学 7 个单词完成今日目标 | 今天又没完成 |
| 错误 | 还不熟，稍后再复习一次 | 回答错误 |
| 成就 | 已连续学习 12 天 | 你击败了所有懒惰的人 |
| 空状态 | 收藏的单词会出现在这里 | 暂无数据 |

数字反馈要真实、可解释。类似“超越 92% 的学习者”的表达只有在存在可靠统计口径时使用。

## 7. 状态与可访问性

每个可交互组件至少定义以下状态：默认、按下、禁用、选中；涉及网络请求时增加加载、成功和失败状态。

- 正文与背景对比度建议达到 WCAG AA。
- 可点击区域不小于 `88rpx × 88rpx`。
- 正确/错误、认识/不认识不能只依赖绿色和红色区分。
- 图表提供数字或文字摘要。
- 图片与纯图标按钮提供 `aria-label`。
- 适配顶部状态栏、底部安全区和系统字体放大。
- 弱网或请求失败时保留用户已完成的操作，并提供重试入口。

## 8. 当前实现的一致性问题

以下内容是基于现有源码的后续统一建议，不影响当前 MVP 的视觉方向：

1. **图标体系混用**：页面同时使用 Tab 图片、字符符号和 Emoji。建议统一为同一套线性图标。
2. **设计 Token 尚未完全收口**：部分页面直接写入相近但不同的绿、灰与圆角值。建议统一到全局变量。
3. **字体权重存在非标准值**：例如 `font-weight: 750`，不同设备渲染可能不一致，建议使用 `700` 或 `800`。
4. **按钮状态不完整**：挑战页下一题按钮尚未表现禁用、正确、错误和提交中状态。
5. **功能入口与 MVP 范围不完全一致**：底部“复习、词库”入口需要补齐页面，或在正式发布前调整信息架构。
6. **字符图标可访问性有限**：收藏、发音、返回、分享等应改用标准图标并补充可访问名称。
7. **数据可信度需要产品规则支撑**：周报中的百分位表达需明确数据来源或改为用户自己的环比结果。

## 9. 新页面验收清单

### 视觉

- [ ] 页面使用米白底色与统一的品牌绿色板。
- [ ] 左右边距为 `32rpx`，主要间距来自统一刻度。
- [ ] 圆角、阴影和卡片样式与现有页面一致。
- [ ] 页面只有一个最高优先级主操作。
- [ ] 图标来自统一图标体系，没有临时 Emoji 功能图标。

### 交互

- [ ] 所有按钮具备按下与禁用状态。
- [ ] 加载、空、错误、弱网和成功状态均有设计。
- [ ] 返回路径清晰，底部安全区适配正常。
- [ ] 关键操作有即时、克制的反馈。

### 内容与可访问性

- [ ] 文案简短、鼓励、可执行，不制造羞耻或焦虑。
- [ ] 颜色不是表达状态的唯一方式。
- [ ] 点击热区不小于 `88rpx × 88rpx`。
- [ ] 图标按钮有可访问名称，图表有文字摘要。
- [ ] 数据口径真实且可解释。

## 10. 推荐的全局 Token

后续重构时，可将常用值集中到 `app.wxss`：

```css
page {
  --color-brand: #087443;
  --color-brand-dark: #075332;
  --color-brand-soft: #e9f5ec;
  --color-accent: #ffd452;
  --color-text: #121b17;
  --color-text-muted: #7b817e;
  --color-bg: #f8f7f2;
  --color-surface: #ffffff;
  --color-danger: #d73a32;
  --color-danger-soft: #fff0ec;
  --color-divider: #e2e1dd;

  --space-1: 8rpx;
  --space-2: 16rpx;
  --space-3: 24rpx;
  --space-4: 32rpx;
  --space-5: 48rpx;
  --space-6: 64rpx;

  --radius-card: 30rpx;
  --radius-control: 26rpx;
  --radius-button: 56rpx;
  --shadow-card: 0 10rpx 30rpx rgba(59, 58, 48, .09);
  --shadow-action: 0 12rpx 26rpx rgba(0, 105, 59, .20);
}
```

这份 Token 是对现有实现的归一化，不要求一次性重写页面。新增页面应优先使用，旧页面可在后续迭代中逐步迁移。
