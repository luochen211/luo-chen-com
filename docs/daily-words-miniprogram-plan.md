# 每日单词小程序产品与开发规划

## 1. 产品定位

面向希望每天用 5–10 分钟稳定积累词汇的用户。首版不追求大而全，只做完整的最小学习闭环：看到今日目标 → 学习单词 → 标记掌握程度 → 答题巩固 → 查看周报。

## 2. MVP 范围

- 首页：连续学习、今日进度、目标修改、学习入口。
- 学习：单词、音标、释义、例句、收藏、认识/不认识。
- 挑战：四选一、答题进度、选项反馈、完成后进入周报。
- 周报：七日学习量、掌握数、待复习数、正确率、连续天数和分享。

## 3. 信息架构与路由

| 页面 | 路径 | 主要任务 |
| --- | --- | --- |
| 首页 | `pages/home/home` | 了解今日状态并开始学习 |
| 学习 | `pages/study/study` | 学习和标记单词 |
| 挑战 | `pages/quiz/quiz` | 完成词义巩固 |
| 周报 | `pages/report/report` | 查看成果和分享 |

## 4. 数据模型

- `users`：`openid`、每日目标、连续天数、最后学习日。
- `words`：单词、音标、释义、例句、音频地址、难度和词书 ID。
- `learning_records`：用户、单词、学习时间、结果、复习次数、下次复习时间。
- `quiz_records`：题目、用户选项、是否正确、作答耗时。

## 5. 技术方案

当前采用原生小程序（WXML + WXSS + JavaScript），避免 MVP 阶段引入额外跨端框架。后端建议优先使用微信云开发：云数据库保存词库与学习记录，云函数计算连续天数和周报，云存储保存发音音频。

## 6. 迭代计划

### 第 1 阶段：可用 MVP（1 周）

完成真实词库接入、微信登录、学习记录落库、发音播放和基础埋点。验收标准是用户可跨天保留进度，且完整走通一次学习闭环。

### 第 2 阶段：复习系统（1 周）

基于简化 SM-2 间隔重复算法生成待复习队列，增加错词本、收藏列表和词书切换。

### 第 3 阶段：增长与优化（1 周）

增加周报分享卡、学习提醒、新手引导、无障碍标签、弱网缓存和性能优化。

## 7. 核心指标

- 激活：新用户首次进入后完成至少 5 个单词的比例。
- 留存：D1 / D7 学习留存率。
- 完成：每日目标完成率和单次学习中途退出率。
- 学习效果：7 天内复习正确率和错词再错率。

## 8. 上线前清单

配置正式 AppID、隐私保护指引与用户协议；确认词库、例句和音频版权；在 iPhone SE、常见全面屏和 Android 机型上验证安全区；补齐空状态、异常状态和断网提示；完成体验版提审和真机灰度测试。
