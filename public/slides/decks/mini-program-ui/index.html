06 / 17 · WARNING
MINI PROGRAM UI
WARNING

六个常见坑

先把这些坑避开，页面质感通常会立刻上一个台阶。

01

emoji 当图标

表情风格不可控，很难统一产品气质

02

蓝紫渐变滥用

看似科技感，实际容易廉价且抢信息

03

页面过空

留白没有服务层级，只剩空

04

页面过满

拥挤也没有服务效率

05

模块承接太多

一屏想解决所有问题，结果没有主线

06

入口不明显

用户每犹豫一次，都会增加流失

07 / 17 · STANDARD
MINI PROGRAM UI
STANDARD

真的好看是四件事同时成立

— 01 / SYSTEM

统一

颜色、图标、间距、图片风格不互相打架。

— 02 / READ

清楚

用户能立刻分辨标题、重点和下一步。

— 03 / FEEL

舒服

空间、字重、对齐让阅读不费劲。

— 04 / ACT

顺手

入口好找、按钮明显、反馈及时。

08 / 17 · COLOR

克制

PRIMARY
1

NEUTRAL
80%

GRADIENT
慎用

09 / 17 · LAYOUT
MINI PROGRAM UI
LAYOUT

02 排版先分层

小程序页面空间很窄，层级比花样更重要。

  1. 01先说清楚：这是什么
  2. 02再突出：重点在哪里
  3. 03最后收束：下一步去哪

层级不是装饰，是把一张长页面变成一条可理解的路径。

12 / 17 · WORKFLOW
START SIMPLE
METHOD

没有灵感
就先复刻

别从空白画布开始。审美不是凭空创造，而是先看懂成熟页面为什么成立。

参考不是答案，是你的起跑线
13 / 17 · WORKFLOW
MINI PROGRAM UI
WORKFLOW

四步把审美落成页面

01 / REFERENCE

找同类

只看同场景、同任务、同信息密度的成熟产品。

02 / STRUCTURE

拆顺序

标出用户先看什么、再看什么、最后点哪里。

03 / FIRST SCREEN

做一屏

先把首屏做到可信，再向下扩展完整页面。

04 / ITERATE

反复改

按统一、清楚、舒服、顺手逐项检查。

15 / 17 · PRACTICE
MINI PROGRAM UI
DO NOT

只说
高级一点

这种需求太抽象，AI 和团队都不知道你要的目标画面是什么。

DO

给页面
给约束

做一个宠物服务小程序首页：
暖白背景，蓝色只用于主按钮；
首屏依次展示定位、核心服务、预约入口；
卡片圆角统一为 16px，图标使用同一套线性风格；
参考 Apple Store 的信息克制感。
CONCLUSION

审美不是
看出来的

它要靠重复实践形成判断。

01

多看

建立自己的参考库，知道什么是及格线。

02

多模仿

把好页面拆成结构、比例、颜色和状态。

03

多改

按统一、清楚、舒服、顺手反复打磨。
